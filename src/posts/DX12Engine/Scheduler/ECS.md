# ECS::Registry（实体组件系统）

## 1. 定位与职责

ECS::Registry 是 **实体组件系统的核心容器**，基于 EnTT 实现，管理所有游戏实体及其组件。

- **上游依赖**：无（纯数据层）
- **下游服务**：FrameDriver（System 调度）、CullingSystem（可见性查询）

### 核心职责

| 职责 | 说明 |
|:----|:-----|
| **实体管理** | 创建/销毁实体，分配唯一 ID |
| **组件管理** | 添加/移除/查询组件，组件池化 |
| **System 注册** | 注册 System 到指定执行阶段 |
| **视图查询** | 按组件组合查询实体集 |

---

## 2. 组件分类

| 组件类别 | 包含 | 用途 |
|:---------|:-----|:-----|
| Transform | 位置、旋转、缩放 | 空间变换 |
| Render | 网格句柄、材质句柄、可见性 | 渲染数据 |
| Light | 光源类型、颜色、强度、范围 | 光照计算 |
| Camera | 投影参数、视口 | 视图投影 |
| Animation | 骨骼动画状态、播放控制器 | 蒙皮动画 |
| Tag | 静态/动态/编辑器标记 | 逻辑分类 |
| Water | 水面参数 | 水体渲染 |
| Grid | 网格参数 | 网格体渲染 |
| ReflectionProbe | 探针位置、捕获参数 | 反射探针 |

---

## 3. System 执行阶段

System 按阶段注册，每个阶段在帧循环的固定时间点执行：

| 阶段 | 执行顺序 | 典型用途 |
|:-----|:---------|:---------|
| EarlyUpdate | 1 | 输入处理、摄像机更新 |
| Update | 2 | 游戏逻辑更新 |
| LateUpdate | 3 | 逻辑后处理 |
| PreCulling | 4 | 剔除前准备 |
| PostCulling | 5 | 剔除后处理 |
| SceneDataUpload | 6 | 场景数据上传 GPU |
| PreRender | 7 | 渲染前准备 |
| Render | 8 | 命令录制 |

---

## 4. 设计原则

| 原则 | 说明 |
|:----|:-----|
| **数据驱动** | System 不持有状态，仅操作组件数据 |
| **阶段隔离** | 不同阶段的 System 不互相依赖，通过组件数据通信 |
| **多线程安全** | System 中不能进行内存分配，渲染 System 只能录制命令 |
| **高频连续→AlwaysRun** | 每帧执行的 System 注册为 AlwaysRun |
| **低频离散→WithMessage** | 事件触发的 System 通过消息激活 |
