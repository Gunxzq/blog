# FrameDriver（帧驱动器）

## 1. 定位与职责

FrameDriver 是 **帧循环调度核心**，负责驱动每帧的多个执行阶段，协调 System 的执行顺序。

- **上游依赖**：ECS::Registry（System 注册）、D3D12DeviceContext（命令执行）
- **下游服务**：为所有 System 提供执行环境，管理渲染阶段提交

### 核心职责

| 职责 | 说明 |
|:----|:-----|
| **帧循环驱动** | 每帧按阶段顺序执行：EarlyUpdate→Update→LateUpdate→Render |
| **System 调度** | 从消息桶构建本帧任务图（DAG）并执行 |
| **渲染阶段管理** | 收集按阶段提交的命令列表，统一执行 |
| **回调注册** | 提供帧同步回调、即时回调、场景数据上传回调 |
| **帧统计** | 提供帧号、任务数、CPU/GPU 时间统计 |

---

## 2. 每帧执行流程

```
(1) 消息处理：ProcessMessages + Input 更新
(2) 任务图构建：从消息桶构建本帧任务图
(3) 即时回调：零延迟路径（摄像机、UI）
(4) 逻辑阶段：EarlyUpdate → Update → LateUpdate
(5) 剔除阶段：PreCulling → PostCulling
(6) 场景数据上传：PostCulling 之后、PreRender 之前
(7) 渲染阶段：提交各阶段渲染命令
(8) 帧同步：L4 层多缓冲交换回调
(9) 帧率控制：等待目标帧率
```

---

## 3. 回调系统

FrameDriver 提供三种回调注册机制：

| 回调类型 | 执行时机 | 用途 |
|:---------|:---------|:-----|
| **帧同步回调** | 逻辑帧结束、渲染帧开始前 | L4 层多缓冲交换 |
| **即时回调** | 任务图执行前 | 摄像机、UI 等零延迟路径 |
| **场景数据上传回调** | PostCulling 后、PreRender 前 | 探针/光源/地形数据上传 |

---

## 4. 设计原则

| 原则 | 说明 |
|:----|:-----|
| **前台不依赖后台** | 前台每帧从消息构建任务图，不依赖后台状态 |
| **阶段顺序固定** | 执行阶段顺序不可变，各 System 按阶段注册 |
| **渲染命令批量提交** | 同一渲染阶段的命令列表批量提交，不逐条提交 |
| **帧率可配置** | 支持设置目标帧率，空闲时等待 |
