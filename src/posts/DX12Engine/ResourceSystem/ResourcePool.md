# RenderTargetPool & DepthStencilPool（渲染目标池与深度模板池）

## 1. 定位与职责

RenderTargetPool 和 DepthStencilPool 是 **渲染目标和深度模板缓冲的池化管理器**，按需分配和回收，避免频繁创建和销毁。

- **上游依赖**：DescriptorHeapCollection（描述符堆）
- **下游服务**：渲染 Pass（分配临时 RT/DSV）

### 核心职责

| 职责 | 说明 |
|:----|:-----|
| **RT 池化管理** | 按需分配 RT，回收后复用，避免重复创建 |
| **DSV 池化管理** | 按需分配深度模板缓冲 |
| **描述符自动管理** | 自动分配 RTV/DSV 描述符槽位 |

---

## 2. 协作模式

```
资源池角色：
  - 只负责资源生命周期（分配/释放）
  - 不追踪也不重置 GPU resource state
  - 使用资源的 System 必须自己管理 ResourceBarrier

使用规则：
  - 分配 RT → System 负责 Barrier 到所需状态 → 渲染 → Barrier 回原状态 → 归还
  - 不能假设初始状态为 COMMON
  - 必须遵循对称屏障规则（入口转换 + 出口转换）
```

---

## 3. 设计原则

| 原则 | 说明 |
|:----|:-----|
| **状态不追踪** | 资源池不追踪 GPU 状态，由使用者负责 |
| **对称屏障** | 使用者必须完成入口和出口屏障 |
| **按需分配** | 不预分配所有 RT，使用时按需分配 |
| **延迟回收** | 回收后延迟到 fence 完成再复用，防止 GPU 仍在读取 |
