# DebugUI（调试覆盖层）

## 1. 定位与职责

DebugUI 提供运行时的调试信息覆盖层，集成 ImGui 实现开发态的可视化调试。

- **上游依赖**：Window（HWND）、D3D12DeviceContext（命令队列）
- **下游服务**：为开发者提供实时调试面板

### 核心职责

| 职责 | 说明 |
|:----|:-----|
| **调试覆盖层** | 帧率、GPU 时间、内存统计实时显示 |
| **ImGui 集成** | 集成 ImGui Win32 + DX12 后端 |
| **内置面板** | 控制台日志、性能监控、渲染状态查看 |
| **扩展接口** | 允许各模块注册自定义调试面板 |

---

## 2. 初始化流程

```
(1) 初始化 Win32 后端：传入 HWND
(2) 初始化 DX12 后端：传入 Device、CommandQueue、帧数、格式
(3) 配置样式：暗色主题、菜单栏显示
(4) 关联 GameContext：在 CreateContext 阶段完成
(5) 注册到 FrameDriver：自动注册到帧循环的即时回调
```

---

## 3. 设计原则

| 原则 | 说明 |
|:----|:-----|
| **开发态工具** | 仅 Debug/Development 构建启用，Release 完全排除 |
| **零侵入** | 业务代码不依赖 DebugUI，仅通过 Context 可选注入 |
| **独立渲染** | 使用独立的命令列表和描述符堆，不影响主渲染流程 |
