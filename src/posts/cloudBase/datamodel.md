# 数据模型

Cloudbase 的数据模型是对 JSON 数据表的抽象。相较于直接查看底层数据表，数据模型在开发和展示上具有一定的便利性，例如可以直接预览云存储中的图片等资源。

## 数据模型与底层表的区别

在使用时，需要注意数据模型与真实数据库表之间的差异：

1.  **字段类型映射**：
    *   **图片类型**：实际上存储的是字符串（File ID），多个文件 ID 之间通过 `;` 分隔。
    *   **枚举类型**：底层实际上对应的是 `VARCHAR` 类型。
2.  **功能局限性**：
    *   如果需要充分利用云 MySQL 的高级能力（如空间索引），不能完全依赖数据模型创建的字段。因为数据模型目前不支持 `POINT` 等空间数据类型，导致无法使用空间索引功能。
3.  **发布限制**：
    *   如果底层的实际表结构与数据模型定义不匹配，数据模型将无法成功发布。

## 关联关系优势

数据模型可以轻松建立表之间的关联关系。在展示时，关联字段会自动显示为关联模型中的“主展示字段”。

*   **示例**：若 `user_id` 字段关联了 `user` 表，则在界面中该字段会直接显示为用户的名称，而非单纯的 ID，提升了数据的可读性。
