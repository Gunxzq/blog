# ANI 动画格式与分层动画系统 — 2009 年同人游戏的骨骼动画引擎

## 概述

`Ultimate Knight WindomXP` 的动画系统由 **ANI 文件**（`Script.ani`）驱动，支持 29 个骨骼节点的完整骨骼动画，并实现了**上下半身分离动画**的分层混合系统——这在 2009 年的同人游戏中是一个相当高级的 feature。

---

## 1. ANI 文件格式

### 1.1 顶层结构

```
"AN2" 魔术头 (3 bytes)
        │
        ├── "Robo.hod\0" + padding (64 bytes)  ← 关联的骨架文件
        └── padding (0..255B)
        
"HD2" 骨骼通道头 (3 bytes)
        │
        ├── header_size (4B, 0x1E = 30)
        ├── ? (4B)
        ├── ? (4B)
        ├── ? (4B)
        ├── bone_name (64B)  ← 对应 HOD 中的部件名
        │
        ├── 每骨骼关键帧数据:
        │   ├── 帧数 (4B)
        │   ├── 帧率 (4B, float)
        │   └── 每帧:
        │       ├── 位置 (Vector3: 12B)
        │       └── 旋转 (Quaternion: 16B)
        │
        └── ...后续骨骼通道...
```

### 1.2 格式特点

| 特点 | 说明 |
|:-----|:------|
| **"AN2"** | 推测为 "Animation 2"，对应 DX9 时代的动画格式 |
| **"HD2"** | 骨骼通道头，每个骨骼一个通道 |
| **b-one_name** | 64 字节固定长度，对应 HOD 中的部件名 |
| **每帧数据** | 位置 (float3) + 旋转 (quaternion)，无缩放 |
| **帧率** | 每个骨骼通道独立指定帧率 |

## 2. 29 节点骨骼层次

KD-03 机体的完整骨骼层次：

```
root.x
├── Body_d.x
│   └── Body.x
│       ├── Head.x
│       ├── arm1.x → arm2.x → Hand.x
│       │   └── Shield.x → Shield_point.x
│       ├── arm1_r.x → arm2_r.x → Hand_r.x
│       │   ├── gun.x
│       │   ├── sword.x
│       │   └── Weapon_point.x / Weapon_point2.x
│       ├── leg1.x → leg2.x → leg3.x
│       │   ├── Output03.x
│       │   └── Output05.x
│       ├── leg1_r.x → leg2_r.x → leg3_r.x
│       │   └── Output04.x
│       ├── Output01.x
│       ├── Output02.x
│       ├── Output07.x
│       └── Output08.x
```

| 骨骼组 | 节点 | 说明 |
|:-------|:-----|:------|
| **Body_d** | 1 | 身体变形框架 |
| **Body** | 1 | 上半身核心 |
| **左臂链** | arm1 → arm2 → Hand | 3 节点 |
| **右臂链** | arm1_r → arm2_r → Hand_r | 3 节点（持剑/持枪） |
| **左腿链** | leg1 → leg2 → leg3 | 3 节点 |
| **右腿链** | leg1_r → leg2_r → leg3_r | 3 节点 |
| **特效节点** | Output01~08 | 8 节点（武器发射点、推进器喷口等） |
| **其他** | Head, Shield, gun, sword 等 | 7 节点 |

## 4. Tail.dat — 动画事件脚本

每个动画组附带一个 **Tail.dat** 文件（240 字节），内含动画的事件脚本。

### 4.1 二进制头（16 字节）

| 偏移 | 大小 | 类型 | 值 | 说明 |
|:-----|:-----|:-----|:----|:------|
| 0x00 | 4 | uint32 | 1 | 版本/标签 |
| 0x04 | 4 | uint32 | 8 | 条目数 |
| 0x08 | 4 | float | 0.1 | 帧间隔时间（秒）→ 10 FPS |
| 0x0C | 4 | uint32 | 224 | 脚本数据大小 |

### 4.2 内嵌脚本语言

Tail.dat 内嵌 **SPT 格式脚本**，定义了动画播放期间的事件序列：

```
'ShiftJIS_注释' CRLF
SPT_命令序列 CRLF
ENDIF; CRLF
```

**解码示例（01 组 - 立ち/Stand）：**
```
'下半身のみ'       ← 仅下半身动作
IF(@int[151] ,==, 0);
  Move=(0,STOP,0);    ← 停止移动
ENDIF;

'上半身のみ'       ← 仅上半身动作
IF(@int[151] ,==, 1);
  LockArm1Target(-1,-1);
  LockArm2Target(-1,-1);
  LockBodyUpTarget(-1,-1);
  LockBodyDownTarget(-1,-1);
ENDIF;
```

## 5. 分层动画系统（核心设计）

### 5.1 `@int[151]` 状态标志

`@int[151]` 是控制**上下半身分离动画**的核心状态变量：

| @int[151] | 含义 | 行为 |
|-----------|------|------|
| 0 | 下半身主导 | 上体自由（用于行走中攻击） |
| 1 | 上半身主导 | 下体锁定（用于站立射击/防御） |

### 5.2 分层动画命令

| 命令 | 格式 | 说明 |
|:-----|:------|:------|
| `LockArm1Target` | `(strength, duration)` | 锁定左臂 IK 目标 |
| `LockArm2Target` | `(strength, duration)` | 锁定右臂 IK 目标 |
| `LockBodyUpTarget` | `(strength, duration)` | 锁定上半身 IK |
| `LockBodyDownTarget` | `(strength, duration)` | 锁定下半身 IK |

参数 `(-1,-1)` 表示解除锁定。

### 5.3 设计意义

在 2009 年的同人游戏中实现分层动画是一个相当大胆的设计决策：

- **上半身**：播放攻击/举枪/防御动画，独立于下半身
- **下半身**：播放行走/冲刺/跳跃动画，独立于上半身
- **混合**：通过 `@int[151]` 状态标志和 `Lock*Target` IK 锁定实现平滑过渡

这意味着玩家可以在**奔跑中射击**、**跳跃中挥剑**、**冲刺中防御**——这些在现代 3A 游戏中习以为常的功能，在 2009 年的同人游戏中已经通过一套精巧的脚本系统实现了。

## 6. ANI 脚本命令全集

### 动画控制

| 命令 | 格式 | 说明 |
|:-----|:------|:------|
| `ChangeAnime` | `(id, flag, ?)` | 切换到指定动画 ID |
| `ChangeWeapon` | `(WEAPON_TYPE)` | 切换武器（GUN/SWORD 等） |
| `AnimeLoop` | `= 0/1` | 动画循环标志 |
| `GoPoseIndex` | `(-1,2)` | 跳转到指定姿势索引 |
| `GoScriptIndex` | `(7)` | 跳转到指定脚本索引 |

### 移动控制

| 命令 | 格式 | 说明 |
|:-----|:------|:------|
| `Move` | `=(x, mode, speed)` | 设置移动 |
| `MoveLock` | `()` | 锁定移动位置 |
| `Force` | `=(x, y, z)` | 施加外力/速度 |
| `BoostDashMode` | `= 0/1` | 冲刺模式 |

### 攻击判定

| 命令 | 格式 | 说明 |
|:-----|:------|:------|
| `ATTACK` | `(dmg, ?, ?, ?)` | 攻击判定框定义 |
| `AttackDelay` | `(frames)` | 攻击判定延迟 |
| `AttackDown` | `= value` | 击倒积蓄值 |
| `AttackFlag` | `= bits` | 攻击属性标志 |
| `AttackForce` | `= value` | 攻击击退力 |
| `AttackPow` | `= value` | 攻击威力系数 |

### 特效/状态

| 命令 | 格式 | 说明 |
|:-----|:------|:------|

## 7. 社区工具链

| 工具 | 功能 |
|:-----|:------|
| **ANI Tool V1.03** | ANI 反汇编/编辑（分解/打包 .ani） |
| **WindomXP Animations Editor (β0.6)** | HOD 编辑器 + 动画预览 |
| **Mecha Editor for Windom 2.008** | 综合编辑器（ani/hod/script） |
| **Tail.dat Helper** | Tail.dat 自动解密为可读文本 |

### ANI 编辑工作流

1. 用 **ANI Tool** 将 `Script.ani` 分解为多个文件夹（每个动作一个文件夹）
2. 各文件夹序号对应不同动作（如 folder 60 = subattack 3）
3. 用十六进制编辑器（HxD）打开 `Tail.dat`，编辑攻击参数
4. 编辑完成后用 ANI Tool 重新打包为 `Script.ani`

## 总结

起动战士的动画系统展示了令人印象深刻的工程能力：

- **紧凑的 AN2/HD2 格式**：每个骨骼通道独立存储关键帧，支持独立帧率
- **29 节点骨骼层次**：覆盖了完整的机器人骨架，包括特效节点
- **分层动画系统**：通过 `@int[151]` 状态标志实现上下半身分离动画，这在 2009 年的同人游戏中是顶尖水平
- **Tail.dat 事件脚本**：动画不仅是位移和旋转，还嵌入了完整的攻击判定、特效、音效事件系统
- **活跃的社区工具链**：围绕 ANI 格式形成了完整的编辑、分解、打包工具生态
