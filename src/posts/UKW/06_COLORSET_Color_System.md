# COLORSET 换色系统 — 起动战士的材质索引与配色方案

## 概述

`Ultimate Knight WindomXP` 的 **COLORSET** 系统允许玩家为机体选择不同的配色方案。每台机体最多支持 **9 套配色**，玩家可以在战斗中自由切换。

这个系统的设计巧妙之处在于：**它不修改模型文件，也不创建新的纹理，而是通过 SPT 脚本中的 COLOR SET 命令直接覆盖材质参数**。

---

## 1. COLORSET 语法

```
COLORSET(配色编号, 材质编号, a, r, g, b);
```

| 参数 | 说明 |
|:-----|:------|
| **配色编号** | 1~9，玩家可选配色方案 |
| **材质编号** | 对应 `.x` 文件中 Material 的索引（0-based） |
| **a, r, g, b** | float (0~1)，颜色的 Alpha、红、绿、蓝分量 |

### 规则

- 同一配色可设定多个材质（多行 `COLORSET`）
- 未指定的材质保留原色
- 配色编号从 1 开始，最多 9 套

---

## 2. 材质索引映射

机体每个部件（`.x` 文件）包含多个 Material 块，按出现顺序编号（0-based）：

```
COLORSET(1, 0, 1.0, 0.8, 0.2, 0.1);  ← 配色1: 材质0 → 暗红色
COLORSET(1, 1, 1.0, 0.6, 0.6, 0.6);  ← 配色1: 材质1 → 浅灰色
COLORSET(1, 3, 1.0, 0.3, 0.3, 0.3);  ← 配色1: 材质3 → 深灰色
```

### 典型材质分配

| 材质索引 | 部位 | 说明 |
|:--------|:-----|:------|
| 0 | 主装甲色 | 机体主要色调 |
| 1 | 副装甲色 | 辅助色调 |
| 3 | 关节/内构色 | 骨架、关节、内部结构 |
| 4 | 细节/点缀色 | 警示标记、线条等 |

---

## 3. 与 .x 材质的关系

`.x` 文件中的 Material 块包含：

| 字段 | HLSL 类型 | 说明 |
|:-----|:-----------|:------|
| `faceColor` | ColorRGBA | 漫反射颜色（4 floats） |
| `power` | float | 高光指数 |
| `specularColor` | ColorRGB | 高光颜色（3 floats） |
| `emissiveColor` | ColorRGB | 自发光颜色（3 floats） |
| `TextureFilename` | string | 纹理文件名 |

COLORSET 的 `(a, r, g, b)` 参数直接覆盖 `faceColor` 字段，其他字段（`power`、`specularColor`、`emissiveColor`、`TextureFilename`）保持不变。

---

## 4. 纯色材质策略

分析显示，**KD-03 等基础机体完全依赖纯色材质**，没有纹理贴图：

| 机体 | 纹理 | 材质策略 |
|:-----|:------|:---------|
| **KD-03** | 无 `tex.png` | 纯色材质，仅靠 `.x` Material 的 `faceColor`/`specularColor`/`power` |
| **KD-04~08** | 有 `tex.png` (163KB) | 图集纹理，但同样是简单色块 |

这意味着 COLORSET 系统对于无纹理的机体来说就是**唯一的着色手段**——通过调整 4 个材质通道的 RGBA 值，实现完全不同的视觉效果。

---

## 5. 玩家配色方案示例

假设一台机体的默认配色为蓝色主装甲 + 白色副装甲 + 灰色关节，COLORSET 可以定义多套变体：

| 配色编号 | 材质0（主装甲） | 材质1（副装甲） | 材质3（关节） | 视觉效果 |
|:--------|:---------------|:---------------|:-------------|:---------|
| 1（默认） | 蓝色 (0,0,0.8,1) | 白色 (1,1,1,1) | 灰色 (0.5,0.5,0.5,1) | 标准配色 |
| 2 | 红色 (1,0.8,0,0) | 深灰 (0.3,0.3,0.3,1) | 黑色 (0.1,0.1,0.1,1) | 红黑配色 |
| 3 | 绿色 (0,1,0.2,0) | 浅灰 (0.8,0.8,0.8,1) | 深灰 (0.3,0.3,0.3,1) | 军绿色 |
| 4 | 白色 (1,1,1,1) | 蓝色 (0,0,0.8,1) | 灰色 (0.5,0.5,0.5,1) | 反转配色 |
| 5 | 黑色 (0,0.1,0.1,0.1) | 红色 (1,0.8,0,0) | 金色 (1,0.9,0.2,0) | 暗黑系 |

## 6. 从着色器看材质渲染

从游戏自带的着色器（`Basic.txt`）可以看到材质参数如何参与渲染：

```hlsl
// 漫反射计算
float4 Diffuse = Ambient + Diffuse * max(0, N·L);

// 高光计算
float4 Specular = Specular * pow(max(0, R·E), Power);

// 最终输出（无纹理模式）
float4 FinalColor = Diffuse + Specular;
```

COLORSET 修改的 `faceColor` 直接作为 `Diffuse` 参与光照计算，与 `LightDir`、`Eyepos`、`Power` 等全局参数共同作用，产生最终的视觉效果。

---

## 总结

COLORSET 系统展示了起动战士在材质系统中的**高效设计**：

- **零额外资源开销**：不需要为每套配色创建独立的纹理或模型，只需 4 个 float 值
- **9 套配色方案**：每台机体最多支持 9 套配色，覆盖了常见的玩家需求
- **材质索引映射**：基于 `.x` 文件的 Material 索引，与现有模型格式完全兼容
- **纯色友好**：即使是 KD-03 这种完全无纹理的机体，也能通过 COLORSET 实现丰富的视觉效果

在 2009 年的同人游戏中，这种设计使得 MOD 作者可以轻松地为机体添加新配色——只需要在 SPT 文件中添加几行 `COLORSET` 命令，无需任何 3D 建模工具。
