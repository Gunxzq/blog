# 智能体


## 提示词（沟通语言）

一个精准有效的提示词可以帮助模型生成更符合要求的结果。

1. 角色场景法公式：角色[ 专业身份 ] + 场景[ 情景描述 ]
示例：你是资深的产品经理，正在开发一款AI写作助手，需要设计产品核心功能，要求每个功能都要有具体场景、考虑用户痛点、包含创新点以及注重可行性。
2. 参考示例发公式：参照示例 + 需求描述
示例：你是资深的产品经理，正在开发一款AI写作助手，需要设计产品核心功能，要求每个功能都要有具体场景、考虑用户痛点、包含创新点以及注重可行性。
3. 目标任务法公式：目标[ 量化指标 ] + 任务要求
示例：写一篇800字左右的环保文章，主题是日常生活中如何践行环保，内容要包含具体的事例和可行的方法。


## 工作流（神经网络）

工作流（Workflow）是智能体完成执行任务的步骤和流程。

例如，一个智能体要完成电商订单处理任务，它的工作流可能包括接收订单、验证订单信息、处理支付、发货、物流跟踪等步骤。每个步骤都有明确的输入和输出，智能体按照这个流程依次执行，确保订单能够顺利处理。

通过设计合理的工作流，我们可以让智能体更好地处理复杂的任务，提高工作效率和质量。


## 知识库（记忆中枢）

知识库（Knowledge Base）是智能体存储知识和检索信息的地方。其中包含各种领域的知识、规则、案例等。

知识库既可以是结构化的，也可以是非结构化的。

## 插件（延伸手脚）

插件（plugin）是为智能体扩展功能的模块，让智能体具备更多的能力。通过安装不同的插件，智能体可以适应不同的场景和需求。

例如，一个基础的智能聊天机器人可以通过安装翻译插件实现多语言翻译功能，或者通过安装图像识别插件实现对图片内容的识别和分析功能。
