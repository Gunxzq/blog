引擎核心架构总览

1. 设计哲学：装配层驱动的能力域

引擎以 Bootstrap（装配层） 为起点，将所有基础设施模块初始化为 能力域（Capability Domain），并通过 GameContext（能力容器） 注入到各业务模块。

┌─────────────────────────────────────────────────────────────┐
│                     Bootstrap (装配层)                       │
│  初始化所有模块 → 填充到 GameContext → 注入 Game/Editor     │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                  GameContext (能力容器)                       │
│  持有所有能力域的访问指针，提供便捷访问方法                   │
└─────────────────────────────────────────────────────────────┘
                            │
              ┌─────────────┼──────────────┐
              ▼             ▼              ▼
        ┌──────────┐ ┌──────────┐ ┌──────────────┐
        │  Game    │ │  Editor  │ │  FrameDriver │
        └──────────┘ └──────────┘ └──────────────┘

D3D12DeviceContext（图形设备上下文）

1. 定位与职责

D3D12DeviceContext 是引擎的 图形设备核心，作为 D3D12 API 的封装层，统一管理设备、交换链、命令队列和同步原语。

  • 上游依赖：依赖 Window 提供 HWND 句柄，依赖 ConfigManager 获取渲染参数
  • 下游服务：为所有渲染模块提供设备指针、命令队列、格式查询

核心职责

职责 说明
设备管理 创建 D3D12 设备、调试层、功能级别检测
交换链管理 创建/重建交换链、后台缓冲区管理
命令执行 提供命令队列提交、围栏同步（FlushCommandQueue）
视口管理 维护默认视口和裁剪矩形
格式查询 提供后台缓冲区格式、深度格式、MSAA 状态、描述符大小

DebugUI（调试覆盖层）

1. 定位与职责

DebugUI 提供运行时的调试信息覆盖层，集成 ImGui 实现开发态的可视化调试。

  • 上游依赖：Window（HWND）、D3D12DeviceContext（命令队列）
  • 下游服务：为开发者提供实时调试面板

核心职责

职责 说明
调试覆盖层 帧率、GPU 时间、内存统计实时显示
ImGui 集成 集成 ImGui Win32 + DX12 后端
内置面板 控制台日志、性能监控、渲染状态查看
扩展接口 允许各模块注册自定义调试面板

GameTimer（游戏计时器）

1. 定位与职责

GameTimer 提供高精度计时能力，是帧循环的"心跳"来源。

  • 上游依赖：无（纯工具类）
  • 下游服务：FrameDriver（计算 DeltaTime）、动画系统、物理模拟

核心职责

职责 说明
高精度计时 基于 QueryPerformanceCounter 提供微秒级计时
DeltaTime 计算 每帧帧间隔时间，用于逻辑更新
帧率统计 实时 FPS 计算
时间缩放 支持游戏时间缩放（慢动作/加速）

AmbientOcclusionManager（环境光遮蔽）

1. 定位与职责

AmbientOcclusionManager 提供 屏幕空间环境光遮蔽（SSAO） 效果，增强场景深度感和细节。

  • 上游依赖：D3D12DeviceContext（设备）、DescriptorHeapCollection（描述符堆）
  • 下游服务：光照渲染 Pass（提供 AO 因子）

核心职责

职责 说明
SSAO 计算 基于深度缓冲的屏幕空间 AO 计算
模糊处理 对 AO 结果进行双边滤波，消除噪点
尺寸管理 支持半分辨率 AO 以提高性能

CameraManager（摄像机管理器）

1. 定位与职责

CameraManager 管理所有摄像机的 视图投影矩阵 和 视锥体数据，是渲染管线的"眼睛"。

  • 上游依赖：无（独立单例）
  • 下游服务：CullingSystem（视锥体裁剪）、渲染 System（视图/投影矩阵）

核心职责

职责 说明
摄像机参数管理 位置、朝向、FOV、近远平面
视图/投影矩阵计算 每帧更新 ViewMatrix、ProjMatrix
视锥体数据 提供视锥体 6 个平面用于剔除
多摄像机支持 主摄像机 + 调试摄像机 + 反射探针摄像机

CullingSystem（可见性剔除系统）

1. 定位与职责

CullingSystem 负责 可见性剔除，减少不必要的渲染调用。

  • 上游依赖：ECS::Registry（实体查询）、CameraManager（视锥体）
  • 下游服务：渲染 System（提供可见实体列表）

核心职责

职责 说明
视锥体裁剪 剔除不在摄像机视锥体内的实体
可见集标记 标记可见/不可见实体，供渲染 System 使用
遮挡查询 可选：硬件遮挡查询（未来扩展）

FrameResourceManager（帧资源管理器）

1. 定位与职责

FrameResourceManager 管理每帧临时资源，是 GPU 双缓冲/三缓冲机制的核心。

  • 上游依赖：D3D12DeviceContext（设备指针）、DescriptorHeapCollection（描述符堆）
  • 下游服务：所有渲染 System（分配临时缓冲区、PassCB）

核心职责

职责 说明
环形缓冲区管理 每种资源类型独立 RingBuffer，支持动态分配
Pass Constants 固定位置每帧覆盖的常量缓冲区
临时描述符 从 DescriptorHeapCollection 分配临时 SRV 槽位
帧轮换 3 帧轮换，每帧独立资源，配合围栏同步

LODSystem（层次细节系统）

1. 定位与职责

LODSystem 根据实体与摄像机的距离，自动选择合适精度的网格模型。

  • 上游依赖：CameraManager（视点位置）、GeometryResourceManager（网格数据）
  • 下游服务：渲染 System（提供 LOD 级别信息）

核心职责

职责 说明
LOD 级别选择 根据距离切换 LOD 0/1/2/3
LOD 配置 每个级别对应的距离阈值、屏幕占比
与 Culling 协作 在剔除后执行，仅对可见实体计算 LOD

ReflectionProbeManager（反射探针管理器）

1. 定位与职责

ReflectionProbeManager 管理 反射探针 的捕获和采样，提供环境反射效果。

  • 上游依赖：D3D12DeviceContext（设备）、DescriptorHeapCollection（描述符堆）
  • 下游服务：渲染管线（提供反射 Cubemap Array）

核心职责

职责 说明
探针捕获 在探针位置渲染 6 个面的场景到 Cubemap
Cubemap Array 管理 管理多个探针的 Cubemap Array 纹理
LOD 生成 自动生成 Cubemap 的 Mip 链用于粗糙度采样
探针插值 实体位于多个探针之间时进行插值混合
